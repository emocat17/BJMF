## Web 版自动签到系统开发路线图（进度更新：2026-03-02）

### 阶段一：总体设计与项目结构 ✅（已完成）
- **目标**: 在不影响现有脚本的前提下，新增一个独立的 Web 版自动签到系统。
- **任务**:
  - **确定技术栈**: 后端使用 `Flask` + `APScheduler` + `SQLite`，前端使用原生 HTML/CSS/JS（可按需使用轻量 CSS 框架），采用 REST API 通信。
  - **目录规划**: 在仓库根目录下新增 `web_signin/` 目录，内部再划分：
    - `app/`：后端 Flask 应用代码。
    - `static/`：前端静态资源（HTML/CSS/JS）。
    - `templates/`：如需使用 Flask 模板引擎时的模板文件。
    - `db/`：SQLite 数据库文件（例如 `web_signin.db`）。
    - `Dockerfile`、`requirements.txt` 等部署相关文件。
  - **需求细化**: 明确用户流程（扫码登录 → 拉取账号信息 → 配置签到任务 → 后台自动执行）、管理员权限（查看/修改任务、日志）和安全边界（cookie 不在前端展示且不可编辑）。

### 阶段二：数据库与数据模型设计 ✅（已完成）
- **目标**: 使用 SQLite 管理签到任务和执行日志，支持多时间点、启用/禁用等功能。
- **任务**:
  - **设计任务表 `tasks`**（建议字段）：
    - `id`（主键、自增）
    - `name`（学生姓名，显示用）
    - `class_id`（班级 ID 或名称）
    - `cookie`（登录凭据，仅后端使用）
    - `lat`、`lng`、`acc`（经纬度与精度）
    - `wx_key`（微信通知 Key，原 `WXKey`）
    - `qq_key`（QQ 通知 Key，原 `QmsgKEY`）
    - `times`（JSON 字符串，存每天执行的多个时间点，例如 `["07:30","12:30"]`）
    - `enabled`（是否启用该任务）
    - `created_at`、`updated_at`
  - **设计日志表 `task_logs`**：
    - `id`（主键、自增）
    - `task_id`（关联 `tasks.id`）
    - `run_at`（实际执行时间）
    - `status`（如 `success` / `error` / `skipped`）
    - `message`（执行详情或异常信息，建议整体存储一段可读日志文本）
    - **日志内容示例**（结构化为多行文本，方便在前端直接展示）：
      - ===================22:04:55===================
      - =========== 用户和班级信息 ===============
      - 用户姓名: 计专一班-方浩宇
      - 班级名称: 计算机技术1.2.3班男生-王涛
      - 班级码 QCA8QV计算机技术1，2，3班
      - 班级代号: 90619
      - 班级码: QCA8QV
      - 检测到已完成签到: 08:00 已签到
      - ===============================================
      - ===============================================
    - **日志保留策略**：数据库中仅保留最近 7 天的日志记录，定期清理更早的日志。
  - **初始化脚本**: 提供一个初始化数据库的脚本或在 Flask 启动时自动建表逻辑。

### 阶段三：后端核心功能开发（Flask + APScheduler） 🟡（进行中）
- **目标**: 实现扫码登录、任务管理 API 与定时执行逻辑。
- **任务**:
  - **封装现有扫码登录逻辑** ✅ *已完成*：
    - 将 `auto_add_user.py` 中的 `QRCodeLogin` 与 `StudentSession` 抽取为可复用模块（例如 `utils/qr_login.py`），在 Web 后端中导入使用。
  - **扫码登录接口** ✅ *已完成*：
    - `GET /api/qr`：创建一轮扫码登录会话，返回二维码图片 URL 或 base64 以及会话标识（token）。
    - `GET /api/qr/status?token=...`：轮询扫码状态，成功后在后端保存 cookie 并返回用户基础信息（姓名、班级），不将 cookie 透出前端。
  - **任务管理接口（用户侧）** ✅ *已完成*：
    - `POST /api/tasks`：基于「已扫码的会话」创建签到任务，入参包括：经纬度字符串（后端自动解析为 `lat`/`lng`，精度统一为 30）、微信 Key、QQ Key、时间点列表等。
    - `GET /api/tasks/{id}`：返回当前任务详情（不包含 cookie），方便前端展示给用户。
  - **定时执行与 APScheduler 集成** 🟡 *部分完成*：
    - ✅ 服务启动时加载 `enabled=1` 的所有任务，根据 `times` 字段为每个任务注册多个「每天固定时间执行」的 job（见 `scheduler.register_existing_tasks` / `register_task_jobs`）。
    - ✅ 统一使用 `Asia/Shanghai` 时区处理时间与调度。
    - 🟡 封装一个「执行签到」函数：目前 `run_task_job` 已写入占位日志并实现「仅保留最近 7 天日志」清理逻辑，但尚未真正接入现有脚本中的签到逻辑（`utils/attendance.py` 等）。
    - 🔲 在任务新增、修改、禁用时动态更新 APScheduler 中对应 job：当前仅在任务创建时注册 job，尚未提供任务修改/禁用接口。

### 阶段四：前端交互页面开发 ✅（已完成）
- **目标**: 提供一个用户友好、匿名的前端页面，用于创建与查看个人签到任务。
- **任务**:
  - **基础页面结构**:
    - 首页展示一个「步骤向导」式界面：步骤 1 扫码 → 步骤 2 填写与确认 → 步骤 3 提交并提示结果。
  - **扫码登录交互**:
    - 调用 `GET /api/qr` 获取二维码信息，在页面中展示二维码（img 标签或 canvas）。
    - 前端定时轮询 `GET /api/qr/status`，侦测扫码是否成功、获取姓名和班级信息。
    - 扫码成功后，不显示 cookie，仅展示「已获取账号：姓名（班级）」之类视觉反馈。
  - **任务配置表单**:
    - 只读字段：姓名、班级。
    - 可编辑字段：
      - 经纬度（单行文本框，可直接粘贴整段坐标字符串，后端自动解析；精度固定 30），可提供「使用浏览器定位」按钮（调用 HTML5 geolocation）。
      - 微信通知 Key（WXKey）与 QQ 通知 Key（QmsgKEY），允许为空。
      - 每天执行时间点：支持用户输入一个时间点后点击「添加」按钮形成时间列表。
    - 提交按钮：调用 `POST /api/tasks` 新建任务，返回成功提示与任务概要。
  - **样式与体验**:
    - 使用简洁的卡片式布局，适配 PC 和手机浏览器。
    - 通过适量配色与状态提示（loading、成功、错误）提升可用性。

### 阶段五：管理员后台与运维功能 ⏳（未开始 / 仅预留模型）
- **目标**: 为维护者提供简单的后台页面，用于查看与管理所有任务与日志，权限模型尽量简化。
- **任务**:
  - **权限模型（按首次访问设置秘钥的方式实现）**:
    - 第一次有人访问后台功能时，如果数据库中还没有管理员 Key：
      - 前端弹出设置 Key 的页面，引导用户设置一串管理员秘钥（长度和复杂度给出提示即可）。
      - 后端将该 Key 进行加密/哈希后存入数据库（如在一个 `settings` 表中存储 `admin_key_hash`）。
    - 后续访问后台时：
      - 网页右上角始终展示一个「齿轮/盾牌」小图标，点击后弹出输入 Key 的对话框。
      - 前端将用户输入的 Key 发给后端，后端对比加密/哈希结果是否一致，一致则视为管理员登录成功。
      - 无需复杂的多用户体系，也无需额外会话管理，采用短期 token 或简单 Cookie 会话保持登录状态即可。
  - **后台 API 接口**:
    - `POST /api/admin/login`：提交管理员 Key，后端校验通过后返回一个短期 admin token（可以是简单随机字符串或自签 JWT）。
    - `GET /api/admin/tasks`：分页返回所有任务列表，可按姓名、班级、启用状态等筛选。
    - `PATCH /api/admin/tasks/{id}`：修改任务配置（地理信息、时间点、通知 Key、启用/禁用等）。
    - `GET /api/admin/logs?task_id=...`：返回指定任务的执行日志。
